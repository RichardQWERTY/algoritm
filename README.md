# Инструкция по использованию ER-диаграммы

## Файлы в этой папке

1. **ER-диаграмма.md** - подробное текстовое описание всех моделей данных
2. **models.py** - готовый код моделей Django для копирования в проект
3. **database_schema.dbml** - схема базы данных для визуализации
4. **README.md** - этот файл с инструкциями

## Как визуализировать диаграмму

### Вариант 1: dbdiagram.io (рекомендуется)
1. Откройте https://dbdiagram.io/
2. Скопируйте содержимое файла `database_schema.dbml`
3. Вставьте в редактор на сайте
4. Диаграмма построится автоматически
5. Можно экспортировать в PNG, PDF или SQL

### Вариант 2: draw.io
1. Откройте https://app.diagrams.net/
2. Создайте новую диаграмму
3. Используйте описание из `ER-диаграмма.md` для создания схемы вручную
4. Используйте фигуры "Entity Relation" из библиотеки

### Вариант 3: Ручная схема
Можно нарисовать схему от руки на бумаге, используя описание из `ER-диаграмма.md`

## Описание моделей

### Основные сущности:
- **User** - пользователи системы (встроенная модель Django)
- **Category** - категории товаров
- **Product** - товары в каталоге
- **Order** - заказы покупателей
- **OrderItem** - позиции заказа (связь заказа и товара)
- **Review** - отзывы на товары
- **Cart** - корзина покупок
- **CartItem** - позиции корзины

### Типы связей:

#### One-to-Many (1:N):
- Category → Product (одна категория содержит много товаров)
- User → Product (пользователь может добавить много товаров)
- User → Order (пользователь может сделать много заказов)
- User → Review (пользователь может оставить много отзывов)
- Product → Review (у товара может быть много отзывов)
- Product → OrderItem (товар может быть в разных заказах)
- Product → CartItem (товар может быть в разных корзинах)
- Order → OrderItem (заказ содержит много позиций)
- Cart → CartItem (корзина содержит много позиций)

#### One-to-One (1:1):
- User → Cart (у пользователя одна корзина)

## Следующие шаги

После утверждения ER-диаграммы:

1. Создать Django проект
2. Скопировать код из `models.py` в файл `models.py` вашего приложения
3. Установить Pillow для работы с изображениями: `pip install Pillow`
4. Создать миграции: `python manage.py makemigrations`
5. Применить миграции: `python manage.py migrate`
6. Зарегистрировать модели в `admin.py`

## Особенности реализации

### Валидация:
- Цена товара > 0
- Количество на складе >= 0
- Рейтинг отзыва от 1 до 5
- Один пользователь может оставить только один отзыв на товар

### Индексы для оптимизации:
- Product: (available, category) - для фильтрации доступных товаров
- Order: (user, created_at) - для истории заказов
- Review: (product, created_at) - для получения отзывов

### Методы моделей:
- Product.get_average_rating() - средняя оценка товара
- Product.get_reviews_count() - количество отзывов
- Order.get_total_cost() - общая стоимость заказа
- Cart.get_total_cost() - общая стоимость корзины
- Cart.clear() - очистить корзину

## Уникальная фича проекта

Рекомендуемые фичи для реализации на этапе 7:

1. **Система рекомендаций** - "Похожие товары" на основе категории и отзывов
2. **Wishlist (список желаний)** - пользователь может сохранять товары
3. **Сравнение товаров** - сравнение характеристик нескольких товаров
4. **История просмотров** - отслеживание просмотренных товаров
5. **Уведомления о снижении цены** - подписка на товар
6. **Промокоды и скидки** - система купонов
7. **Рейтинг продавцов** - если товары добавляют разные пользователи
