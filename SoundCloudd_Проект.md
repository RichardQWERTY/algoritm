## 2. Идея

### Что за проект
"**???**" — это веб-приложение(форум) для обмена музыкальными треками, вдохновлённое SoundCloud. Пользователи могут загружать свои аудиофайлы, прослушивать треки других авторов, комментировать и создавать плейлисты.

### Какую проблему решает
- Даёт начинающим музыкантам площадку для публикации своих работ
- Упрощает поиск и обнаружение новой музыки
- Создаёт сообщество единомышленников вокруг музыкального творчества
- Цензура

---

## 3. Анализ аналогов

| Сервис | Сильные стороны | Что берём себе |
|--------|----------------|----------------|
| **SoundCloud** | Простая загрузка, волновая форма, комментарии к моментам трека | Загрузка аудио, плеер с визуализацией, система комментариев |
| **Bandcamp** | Поддержка независимых артистов, продажа музыки | Профиль артиста, возможность лайкать треки |
| **Jamendo** | Бесплатная музыка, лицензирование | Категоризация по жанрам, плейлисты |

---

## 4. MVP — Ключевой функционал (обязательно)

- [ ] **Регистрация и авторизация** пользователей (Django Auth)
- [ ] **Загрузка аудиофайлов** (MP3, WAV) с обложкой и описанием
- [ ] **Страница трека** с встроенным аудиоплеером
- [ ] **Главная страница** с лентой последних треков
- [ ] **Профиль пользователя** с его треками
- [ ] **Лайки и комментарии** к трекам
- [ ] **Поиск** по названиям треков и артистам
- [ ] **Модель данных:**
  - `User` (стандартная Django)
  - `Track` (название, файл, обложка, описание, автор, дата)
  - `Comment` (текст, автор, трек, дата)
  - `Like` (пользователь, трек)

---

## 5. Уникальные фичи (опционально — доп. балл)

- [ ] **Визуализация волны** аудиофайла (Waveform.js)
- [ ] **Комментарии к конкретному моменту** трека (timestamp), комментарии у треку, анон. комментарии 
- [ ] **Плейлисты** — пользователи создают свои подборки
- [ ] **Жанры/теги** — фильтрация по категориям
- [ ] **Статистика прослушиваний** — счётчик для каждого трека
- [ ] **Скачивание треков** (если автор разрешил)
- [ ] **Тёмная тема** оформления

## 6. Технологии

### Backend
- **Django 5.x** — основной фреймворк
- **Django REST Framework** (опционально) — API для будущего расширения
- **SQLite** (для разработки) / **PostgreSQL** (для деплоя) — база данных

### Frontend
- **HTML/CSS/JavaScript** — базовая разметка и стили
- **Bootstrap 5** — адаптивный дизайн
- **Howler.js** или **HTML5 Audio API** — аудиоплеер
- **WaveSurfer.js** (опционально) — визуализация волны

### Дополнительные библиотеки
- `mutagen` — чтение метаданных аудиофайлов
- `Pillow` — обработка изображений (обложек)
- `django-crispy-forms` — красивые формы

### Деплой (опционально)
- **PythonAnywhere** / **Heroku** / **Render** — бесплатный хостинг
- **WhiteNoise** — раздача статики
- **Gunicorn** — WSGI-сервер
