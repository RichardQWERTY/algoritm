# 🎵 SoundCloudd

**Веб-платформа для загрузки и прослушивания музыки**

*Учебная практика 2026*

---

## 📋 О проекте

**SoundCloudd** — веб-приложение для обмена музыкальными треками, вдохновлённое SoundCloud

### Возможности:
- 🎧 Загрузка аудиофайлов
- 🎤 Прослушивание треков
- 💬 Комментирование
- 📝 Создание плейлистов

---

## 🎯 Какую проблему решает

✅ **Для начинающих музыкантов** — площадка для публикации работ

✅ **Для слушателей** — упрощённый поиск и обнаружение новой музыки

✅ **Для сообщества** — объединение единомышленников вокруг музыкального творчества

---

## 📊 Анализ аналогов

| Сервис | Сильные стороны | Что берём себе |
|--------|----------------|----------------|
| **SoundCloud** | Загрузка, волновая форма, комментарии | Плеер с визуализацией, система комментариев |
| **Bandcamp** | Поддержка независимых артистов | Профиль артиста, лайки |
| **Jamendo** | Бесплатная музыка, лицензирование | Категоризация по жанрам, плейлисты |

---

## 🚀 MVP — Ключевой функционал

- 🔐 Регистрация и авторизация (Django Auth)
- 📤 Загрузка аудиофайлов (MP3, WAV) с обложкой
- 🎵 Страница трека с аудиоплеером
- 🏠 Главная страница с лентой треков
- 👤 Профиль пользователя
- ❤️ Лайки и комментарии
- 🔍 Поиск по трекам и артистам

---

## 📦 Модель данных

**User** — стандартная модель Django

**Track** — название, файл, обложка, описание, автор, дата

**Comment** — текст, автор, трек, дата

**Like** — пользователь, трек

---

## ⭐ Уникальные фичи

- 🌊 **Визуализация волны** (Waveform.js)
- ⏱️ **Комментарии к моменту трека** (timestamp)
- 📚 **Плейлисты** — пользовательские подборки
- 🏷️ **Жанры/теги** — фильтрация по категориям
- 📊 **Статистика прослушиваний**
- ⬇️ **Скачивание треков** (по разрешению автора)
- 🌙 **Тёмная тема** оформления

---

## 🛠️ Технологии: Backend

- **Django 5.x** — основной фреймворк
- **Django REST Framework** — API для будущего расширения
- **SQLite** (разработка) / **PostgreSQL** (деплой) — база данных

---

## 🎨 Технологии: Frontend

- **HTML/CSS/JavaScript** — базовая разметка и стили
- **Bootstrap 5** — адаптивный дизайн
- **Howler.js** / **HTML5 Audio API** — аудиоплеер
- **WaveSurfer.js** — визуализация волны (опционально)

---

## 📚 Дополнительные библиотеки

- `mutagen` — чтение метаданных аудиофайлов
- `Pillow` — обработка изображений (обложек)
- `django-crispy-forms` — красивые формы

### Деплой (опционально)
- **PythonAnywhere** / **Heroku** / **Render** — бесплатный хостинг
- **WhiteNoise** — раздача статики
- **Gunicorn** — WSGI-сервер

## 📁 Структура проекта

**soundcloudd/**
1. `manage.py` — управление проектом
2. **soundcloudd/** — основной проект
   - `settings.py` — конфигурация
   - `urls.py` — маршруты
   - `wsgi.py` — WSGI-сервер
3. **tracks/** — приложение треков
   - `models.py` — Track, Comment, Like
   - `views.py` — представления
   - `urls.py` — маршруты приложения
4. **users/** — приложение пользователей
   - `models.py` — профиль пользователя
   - `views.py` — представления
5. **media/** — загруженные файлы
   - `tracks/` — аудиофайлы
   - `covers/` — обложки
6. **static/** — статические файлы
   - `css/`, `js/`, `img/`
